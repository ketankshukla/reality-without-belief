# The Bug Hunter

## Author
**Finn Ryder** — LitRPG

---

## Genre
LitRPG / System Exploitation / Hacker Progression

---

## Concept

The System that governs Earth is supposed to be perfect. Divine code, unbreakable rules, flawless balance. Every awakened human accepts this as fact.

Zoe Martinez was a QA tester before the awakening. She broke software for a living. And she's noticed something the System's creators didn't expect: bugs. Tiny inconsistencies. Edge cases that don't behave as documented. Exploits that a billion users will eventually stumble upon.

When Zoe reports her first bug to the System — through a mental interface that shouldn't exist — she receives a unique class: Bug Hunter. Her job is to find and report exploits before other players abuse them. In exchange, she gets to keep a small version of each exploit she discovers.

Her class is supposed to help the System. But the more bugs Zoe finds, the more she realizes: the System isn't divine. It's software. And software has developers. Someone wrote this code. Someone is maintaining it. Someone is watching.

The bugs aren't just errors. They're breadcrumbs. And Zoe is about to find out what happens when a player starts asking questions about the code itself.
