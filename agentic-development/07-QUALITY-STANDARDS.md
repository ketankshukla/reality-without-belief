# Quality Standards

## Overview

This document defines the quality standards derived from the **Reality Without Belief** project — the gold standard for AI-generated book projects. All generated content must meet or exceed these standards.

---

## Part 1: Writing Quality

### 1.1 Tone Standards

**The Reality Without Belief Standard:**

| Quality | Description |
|---------|-------------|
| **Inspirational** | Every chapter leaves the reader feeling empowered and capable |
| **Firm** | No hedging, no apologizing, no softening the message |
| **Unapologetic** | The author stands confidently behind every claim |
| **Transformative** | Language emphasizes what becomes POSSIBLE, not what is lost |
| **Direct** | Every sentence earns its place — no filler |

**Tone Calibration Scale:**

```
TOO SOFT                    IDEAL                    TOO HARSH
────────────────────────────────────────────────────────────────
"Maybe consider..."    "This is how it works"    "You're an idiot if..."
"You might find..."    "You will discover..."    "Obviously..."
"Perhaps this could..."  "This changes..."       "Only fools..."
```

### 1.2 Language Standards

**Required Elements:**

- **Em dashes (—)** for rhetorical emphasis and rhythm
- **Active voice** as default (passive only when intentional)
- **Short paragraphs** for impact (3-5 sentences typically)
- **Powerful verbs** over weak verb + adverb combinations
- **Specific examples** over vague generalizations
- **Present tense** for immediate engagement

**Prohibited Elements:**

| Avoid | Replace With |
|-------|--------------|
| "Maybe," "perhaps," "might" | Direct statements |
| "I think," "I believe" | Declarative assertions |
| "This may be uncomfortable" | "This will transform" |
| "Void," "emptiness" | "Clarity," "freedom," "spaciousness" |
| Passive voice (usually) | Active constructions |
| Filler phrases | Nothing — delete them |
| Hedging qualifiers | Confident statements |

**Example Transformation:**

```
BEFORE (weak):
"You might find that perhaps letting go of your beliefs could 
possibly lead to some sense of freedom, though this may be 
uncomfortable for some readers."

AFTER (strong):
"The moment you release belief, freedom arrives. Not someday. 
Not gradually. Immediately. This is what awaits — and it will 
transform everything."
```

### 1.3 Structure Standards

**Chapter Structure Quality Markers:**

| Element | Quality Standard |
|---------|-----------------|
| **Opening** | Hooks within first 2 sentences |
| **Flow** | Each paragraph leads naturally to next |
| **Build** | Complexity increases progressively |
| **Climax** | Clear "aha" moment in each chapter |
| **Bridge** | Final paragraph creates momentum to next chapter |

**Paragraph Standards:**

- First sentence = topic or hook
- Middle sentences = development
- Final sentence = transition or punch
- Average length: 50-100 words
- Maximum length: 150 words

### 1.4 Word Count Standards

| Content Type | Target | Minimum | Maximum |
|--------------|--------|---------|---------|
| Chapter (non-fiction) | 3,000 | 2,500 | 4,500 |
| Chapter (fiction) | 2,500 | 2,000 | 4,000 |
| Introduction | 2,000 | 1,500 | 3,000 |
| Preface | 700 | 500 | 1,000 |
| Prologue | 1,000 | 800 | 1,500 |
| Epilogue | 1,200 | 800 | 2,000 |
| Chapter Summary | 250 | 200 | 350 |
| Book Summary | 800 | 700 | 900 |
| Description | 300 | 200 | 400 |

## Part 2: Content Quality

### 2.1 Front Matter Standards

**Copyright Page:**
- [ ] Current year
- [ ] Correct author name
- [ ] Correct book title
- [ ] Series information (if applicable)
- [ ] Edition information
- [ ] ISBN placeholders
- [ ] Professional formatting

**Dedication:**
- [ ] Genuine and personal
- [ ] Matches book's tone
- [ ] Brief but meaningful
- [ ] No generic phrases

**Epigraph:**
- [ ] Directly relevant to book's themes
- [ ] From credible source
- [ ] Properly attributed
- [ ] Not overused in genre

**Preface:**
- [ ] Establishes author's voice immediately
- [ ] Explains why this book exists
- [ ] Creates personal connection
- [ ] Sets expectations appropriately

**Introduction:**
- [ ] Hooks reader immediately
- [ ] Clearly states book's promise
- [ ] Previews all chapters specifically
- [ ] Creates urgency to continue
- [ ] Includes series preview (if applicable)

**Prologue:**
- [ ] Narrative, not expository
- [ ] Emotionally engaging
- [ ] Relevant to book's themes
- [ ] Bridges to main content

### 2.2 Chapter Standards

**Each Chapter Must:**
- [ ] Have a clear, compelling title
- [ ] Open with a hook (statement, question, or scene)
- [ ] Present one core argument or narrative
- [ ] Include specific examples or evidence
- [ ] Build to a clear insight or climax
- [ ] End with forward momentum
- [ ] Connect to overall book arc
- [ ] Reinforce at least one major theme

**Chapter Quality Rubric:**

| Score | Description |
|-------|-------------|
| **5** | Exceptional — could be standalone essay/story |
| **4** | Strong — accomplishes all goals clearly |
| **3** | Adequate — meets requirements but lacks polish |
| **2** | Weak — missing key elements or unclear |
| **1** | Unacceptable — requires complete rewrite |

**Target:** All chapters score 4 or 5

### 2.3 Back Matter Standards

**Epilogue:**
- [ ] Reflects on the journey completed
- [ ] Synthesizes key insights
- [ ] Points forward (series context)
- [ ] Ends powerfully

**Acknowledgments:**
- [ ] Genuine and specific
- [ ] Includes key influences
- [ ] Professional tone
- [ ] Appropriate length

**About the Author:**
- [ ] Establishes credibility
- [ ] Creates connection
- [ ] Includes relevant details
- [ ] Provides contact/website

**Call to Action:**
- [ ] Clear and specific
- [ ] Appropriate ask (reviews, newsletter)
- [ ] Not pushy
- [ ] Provides easy next step

## Part 3: Image Prompt Quality

### 3.1 Prompt Completeness Checklist

Every image prompt must include:

- [ ] **Dimensions** specified (e.g., 2700×4800 pixels)
- [ ] **Resolution** specified (e.g., 300 DPI)
- [ ] **Color space** specified (e.g., sRGB)
- [ ] **Style** clearly defined (photorealistic, illustrated, etc.)
- [ ] **Subject** described in extreme detail
- [ ] **Composition rules** for text placement
- [ ] **Background** fully described
- [ ] **Text specifications** exactly as they should appear
- [ ] **Typography** (font, size, color, spacing)
- [ ] **Mood/atmosphere** clearly conveyed
- [ ] **Technical requirements** (no watercolor, etc.)

### 3.2 Detail Level Standard

**Reality Without Belief Example (Face Description):**

```
Her face is captured with hyper-real detail — flawless yet 
entirely natural skin with visible pores across her nose and 
cheeks, a scattering of faint freckles across the bridge of 
her nose, the faintest hint of peach fuzz along her jawline 
caught by the light. Her eyebrows are full, naturally shaped, 
with individual hairs visible and slightly feathered at the 
inner corners. Her eyes are wide open and luminous — large, 
striking ice-blue irises with turquoise striations radiating 
outward from the pupil like crystalline fractures, each 
striation a slightly different shade of teal and cerulean. 
Natural catchlights — two sharp, luminous reflections — sit 
in the upper portion of each iris, giving her eyes a deep, 
glistening, almost supernatural clarity.
```

**This level of detail is required for:**
- Human faces
- Key symbolic elements
- Background scenes
- Text placement specifications

### 3.3 Text Positioning Standard

**Required Precision:**

```
Line 1 (exactly 3% from top): "[TEXT]" — [font], [size], 
[color] (#HEX), letter-spacing [X]em, approximately [N]pt 
equivalent.
```

**Zone System:**
- Zone 1 (0-25%): Title, series name, book number
- Zone 2 (25-30%): Subtitle area
- Zone 3 (30-85%): Character/imagery zone (NO text)
- Zone 4 (85-100%): Author name

### 3.4 Consistency Standards

**For Series:**
- Same font family across all covers
- Same color palette
- Same text positioning percentages
- Same decorative elements
- Same style (photorealistic throughout, etc.)
- Same character representation approach

## Part 4: Description Quality

### 4.1 Amazon Description Standards

**Structure Requirements:**
- [ ] Bold hook opening
- [ ] Problem/premise establishment
- [ ] Book positioning statement
- [ ] Benefit bullet list (5-10 items)
- [ ] Target reader description
- [ ] Closing hook
- [ ] Series/author positioning

**Keyword Integration:**
- Primary keyword in first paragraph
- Secondary keywords in benefits
- Natural integration (not stuffed)

**HTML Formatting:**
- `<b>` for emphasis
- `<i>` for titles and key phrases
- `<p>` for paragraph breaks
- `<ul>` and `<li>` for bullet lists
- Clean, valid HTML

### 4.2 Benefit List Standards

**Each Benefit Must:**
- Start with action verb or "Why/How"
- Be specific to book content
- Promise transformation
- Stand alone as compelling
- Not repeat other benefits

**Example Quality Scale:**

```
WEAK: "Learn about belief"
BETTER: "Understand why belief fails"
BEST: "Why belief is not a virtue — and the extraordinary 
       freedom that opens up when you stop treating it as one"
```

## Part 5: Summary Quality

### 5.1 Chapter Summary Standards

**Structure:**
1. Core Argument (1 paragraph)
2. Key Points (4-5 bullets)
3. One-Line Takeaway

**Quality Markers:**
- [ ] Could be spoken in 3 minutes
- [ ] Captures essence without losing depth
- [ ] Each point is distinct
- [ ] Takeaway is memorable

### 5.2 Book Summary Standards (Video Script)

**Quality Markers:**
- [ ] Grabs attention in first 10 seconds
- [ ] Creates emotional engagement
- [ ] Flows naturally when read aloud
- [ ] Builds to compelling conclusion
- [ ] CTA is clear and urgent
- [ ] ~150 words per minute pacing
- [ ] No scene markers or metadata

## Part 6: Audiobook Quality

### 6.1 Script Preparation Standards

**Text Cleaning Requirements:**
- [ ] Numbers written out ("fifty" not "50")
- [ ] Abbreviations expanded ("Doctor" not "Dr.")
- [ ] Acronyms spelled phonetically or expanded
- [ ] Em dashes replaced with [pause] markers
- [ ] Markdown formatting removed
- [ ] Pronunciation guides for unusual words

### 6.2 Narrator Notes Standards

**Must Include:**
- Overall tone guidance
- Pacing recommendations
- Emphasis patterns
- Character voices (if fiction)
- Emotion cues for key passages
- Technical pronunciations

---

## Part 7: Consistency Standards

### 7.1 Cross-Content Consistency

**Must Match Across All Files:**
- Book title (exact spelling and capitalization)
- Subtitle (exact wording)
- Author name (exact spelling)
- Series name (if applicable)
- Chapter titles
- Key terminology
- Thematic language

### 7.2 Series Consistency (When Applicable)

**Visual Identity:**
- Color palette
- Font selections
- Image style
- Layout proportions

**Content Continuity:**
- "Where We Have Been" accuracy
- "Coming Next" previews
- Cross-book references
- Terminology consistency

## Part 8: Review Checklist

### Final Quality Gate

Before delivery, verify:

**Content Quality:**
- [ ] All content meets tone standards
- [ ] No placeholder text remaining
- [ ] Word counts within range
- [ ] Cross-references accurate
- [ ] No grammatical errors
- [ ] Consistent terminology

**Structural Quality:**
- [ ] All required files present
- [ ] Correct folder structure
- [ ] Proper file naming
- [ ] README updated

**Asset Quality:**
- [ ] Image prompts complete
- [ ] Text specifications exact
- [ ] Descriptions HTML-formatted
- [ ] Keywords integrated

**Technical Quality:**
- [ ] Markdown formatting valid
- [ ] HTML valid (for descriptions)
- [ ] No encoding issues
- [ ] Files properly saved
