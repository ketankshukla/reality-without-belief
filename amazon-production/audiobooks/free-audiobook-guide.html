Overview

Your Zero-Cost Strategy

Amazon distributes audiobooks through ACX (Audiobook Creation Exchange) — the platform that feeds both Audible and Amazon. You have two broad zero-cost paths:

✓ Recommended Path
Free TTS (Text-to-Speech) → Free Audio Editing → ACX Upload. Use Microsoft Edge's built-in neural voices or the free edge-tts Python library to generate high-quality AI narration, then clean up the audio in Audacity (free), and export files that meet ACX spec.
⚠ Important Disclosure Note
ACX/Audible currently allows AI-narrated audiobooks but requires you to declare it during submission. Select "This title was narrated by AI" when uploading. Failing to disclose can result in removal.

The guide below covers every step: extracting text from your PDF, generating audio with free tools, meeting ACX technical specs, editing in Audacity, and submitting.

Step 0 — Know Before You Start

ACX Audio Requirements

Every audio file you submit to ACX must meet these technical specifications exactly. Files that fail QC will be rejected.

SpecRequired ValueNotes
FormatMP3No WAV, AAC, or FLAC
Bit Rate192 kbps CBR or higherMust be Constant Bit Rate (CBR), not VBR
Sample Rate44,100 HzStandard CD-quality rate
ChannelsMono or StereoMono is preferred and slightly smaller
Peak Level-3 dBFS or lowerNo clipping allowed
RMS Loudness-23 dBFS to -18 dBFSAverage volume must fall in this range
Noise Floor-60 dBFS or lowerSilence must be very quiet
Room Tone0.5 sec at start & endEach file must open and close with silence
Opening CreditsRequired separate file"Title, written by Author, narrated by…"
Closing CreditsRequired separate fileCopyright, publisher info

Step 2 — End-to-End Workflow

Complete Production Workflow

Phase A: Extract Text from PDF

1
Install pdfplumber or PyMuPDF (free Python libraries) These extract clean text from your PDF preserving layout. If your PDF was created from a Word/InDesign source, text extraction will be clean. If it's a scanned PDF (image-only), you'll need OCR (use Tesseract — free).
2
Extract and clean the text Split by chapter. Remove page headers/footers, figure captions, tables (describe them verbally), and anything that reads awkwardly aloud. Replace em-dashes (—) with pauses using commas or ellipses. Spell out numbers and abbreviations. Remove URLs.
3
Save as individual .txt files per chapter Name them: 00_opening_credits.txt, 01_chapter_1.txt, etc. This maps directly to your final ACX file structure.

Phase B: Generate Audio with edge-tts (Recommended)

1
Install edge-tts Run in your terminal: pip install edge-tts
2
Pick a voice List all available English voices: edge-tts --list-voices | grep en-US
Recommended voices: en-US-GuyNeural (male, authoritative), en-US-JennyNeural (female, warm), en-US-AriaNeural (female, expressive).
3
Generate each chapter The Python script below automates the full batch. It reads all your .txt files and outputs .mp3 files ready for ACX processing.
Step 3 — Automation

Python Script: PDF → Chapter MP3s

Save this as make_audiobook.py in your project folder. It handles PDF extraction, chapter splitting, and audio generation in one pass.

# ───────────────────────────────────────────────────────────────────── # make_audiobook.py # Requirements: pip install edge-tts pdfplumber asyncio # Usage: python make_audiobook.py mybook.pdf output_folder/ # ───────────────────────────────────────────────────────────────────── import asyncio, os, sys, re, pdfplumber import edge_tts # ── SETTINGS ────────────────────────────────────────────────────────── VOICE = "en-US-GuyNeural" # Change to your preferred voice RATE = "+0%" # Speed: "-10%" slower, "+10%" faster PITCH = "+0Hz" # Pitch adjustment # Patterns that signal a new chapter in your PDF CHAPTER_RE = re.compile( r'^(chapter\s+\d+|chapter\s+[ivxlc]+|part\s+\d+|prologue|epilogue|introduction|conclusion)', re.IGNORECASE | re.MULTILINE ) def extract_chapters(pdf_path): """Extract text from PDF and split into chapters.""" full_text = [] with pdfplumber.open(pdf_path) as pdf: for page in pdf.pages: text = page.extract_text() if text: full_text.append(text) raw = "\n".join(full_text) # Split on chapter headings parts = CHAPTER_RE.split(raw) chapters = [] for i in range(1, len(parts), 2): title = parts[i].strip() body = parts[i+1].strip() if i+1 < len(parts) else "" chapters.append((title, body)) # If no chapter headings found, treat whole book as one chapter if not chapters: chapters = [("Full Book", raw)] return chapters def clean_for_audio(text): """Clean text to read better as audio.""" text = re.sub(r'https?://\S+', '', text) # remove URLs text = re.sub(r'\b(\d+)\b', lambda m: m.group(), text) # numbers (expand if needed) text = text.replace('—', ', ') # em-dashes → pauses text = text.replace('–', ', ') # en-dashes → pauses text = re.sub(r'\s{2,}', ' ', text) # collapse whitespace text = re.sub(r'\n{3,}', '\n\n', text) # max 2 blank lines return text.strip() async def text_to_mp3(text, output_path): """Generate MP3 from text using edge-tts.""" communicate = edge_tts.Communicate(text, VOICE, rate=RATE, pitch=PITCH) await communicate.save(output_path) print(f" ✓ Saved: {output_path}") async def main(): if len(sys.argv) < 3: print("Usage: python make_audiobook.py input.pdf output_dir/") sys.exit(1) pdf_path = sys.argv[1] output_dir = sys.argv[2] os.makedirs(output_dir, exist_ok=True) print(f"Extracting text from {pdf_path}...") chapters = extract_chapters(pdf_path) print(f"Found {len(chapters)} chapter(s). Generating audio...") # Opening credits (customize this text) opening = ("Your Book Title. Written by Your Name. " "Narrated by AI. All rights reserved.") opening_path = os.path.join(output_dir, "00_opening_credits.mp3") await text_to_mp3(opening, opening_path) # Chapters for idx, (title, body) in enumerate(chapters, 1): text = clean_for_audio(f"{title}. {body}") fname = f"{idx:02d}_{title[:30].replace(' ','_')}.mp3" fpath = os.path.join(output_dir, fname) print(f"Generating chapter {idx}: {title}") await text_to_mp3(text, fpath) # Closing credits (customize) closing = ("Copyright [Year], Your Name. Published by [Publisher]. " "All rights reserved. For more information visit [website].") closing_path = os.path.join(output_dir, "99_closing_credits.mp3") await text_to_mp3(closing, closing_path) print("\n✅ All audio files generated. Now process in Audacity.") asyncio.run(main())
Step 4 — Audio Processing

Audacity: Meeting ACX Specs

Download Audacity (free, open-source: audacityteam.org) and the ACX Check plugin to verify your files before submission.

Install ACX Check Plugin

Download from wiki.audacityteam.org/wiki/Nyquist_Plug-ins_Reference and place the .ny file in Audacity's plug-in folder. Then: Analyze → ACX Check after each processing step.

Processing Chain for Each Chapter File

1
Import your MP3 File → Import → Audio. Select your chapter .mp3 file.
2
Add 0.5s silence at start and end Click the very start of the track → Generate → Silence → 0.5 seconds. Repeat at the end of the track. This satisfies ACX's "room tone" requirement.
3
Normalize peaks to -3 dBFS Effect → Normalize → Check "Normalize peak amplitude" → set to -3 dB. This ensures no clipping.
4
Adjust RMS loudness to -18 dBFS Effect → Loudness Normalization → set Perceived Loudness (RMS) to -18 LUFS. This brings the average volume into ACX range.
5
Run ACX Check Analyze → ACX Check. All three values (Peak, RMS, Noise Floor) should show green/pass. If noise floor fails, apply: Effect → Noise Reduction (sample a silent section first as the noise profile, then apply at ~12 dB reduction).
6
Export as MP3 at 192 kbps CBR File → Export → Export as MP3 → Options → Bit Rate Mode: Constant, Quality: 192 kbps. Add metadata (title, author, chapter number).
⚡ Audacity Macro for Batch Processing
Use Tools → Macros to record the Normalize + Loudness Normalization + Export steps as a macro. Then run it on all chapter files at once via Apply Macro to Files. This saves hours of manual work.

Step 5 — Publish

ACX Upload & Submission Guide

1
Create your ACX account at acx.com Sign in with your Amazon account. ACX is free to join.
2
Search for your book or add it manually ACX pulls from Amazon's catalog. Search by ASIN (your Kindle or paperback ASIN). If your book is live on Amazon, it should appear.
3
Click "I have this" → "I want to narrate this title" Select "Myself or someone I know" as the narrator option.
4
Declare AI narration ACX will ask how the book was narrated. Select the AI disclosure option. This is required and non-negotiable.
5
Upload your files in order Upload: Opening Credits → Chapter 1 → Chapter 2 → … → Closing Credits. Files must be in sequence. ACX will auto-run QC on each file.
6
Set royalty structure Choose between: Exclusive distribution (40% royalty) — only on Audible/Amazon/iTunes, or Non-exclusive (25% royalty) — you can also sell elsewhere. For KDP authors already in Kindle Unlimited (exclusive), the exclusive ACX rate is usually the right call.
7
Review takes 10–14 business days ACX QC team checks all files. If rejected, they'll specify which file and spec failed. Fix and re-upload only the failed file.
✓ Distribution Tip
Once live on ACX, your audiobook appears on Audible, Amazon, and Apple Books automatically. For wider distribution (Libro.fm, Downpour, etc.), consider also uploading to Findaway Voices or Draft2Digital — both free to use with royalty-split models.
